Encapsulation means binding data (variables) and methods (functions) that operate on that data into a single unit, while restricting direct access to the data from outside.

In Java, this is achieved through:

Private fields (data hiding)

Public getters/setters (controlled access)

Packages (access boundaries)

ğŸ§± How Packages Support Encapsulation

A package in Java is like a folder that groups related classes.
It not only organizes code but also controls visibility between classes.

Packages + access modifiers (private, default, protected, public) together create encapsulation boundaries.

âœ… Example: With Encapsulation (Using Packages + Access Modifiers)
// File: com/company/Employee.java
package com.company;

public class Employee {
    private String name;
    private double salary;

    // Setter (validation included)
    public void setSalary(double salary) {
        if (salary > 0) this.salary = salary;
        else System.out.println("Invalid salary!");
    }

    // Getter
    public double getSalary() {
        return salary;
    }

    public void setName(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}

// File: com/client/Main.java
package com.client;
import com.company.Employee;

public class Main {
    public static void main(String[] args) {
        Employee emp = new Employee();
        emp.setName("Rohit");
        emp.setSalary(-10000);   // Will trigger validation
        System.out.println(emp.getName() + " earns " + emp.getSalary());
    }
}
